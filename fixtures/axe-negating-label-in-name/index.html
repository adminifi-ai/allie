Negating accessible name fixture

This page contains no axe-detectable defect: axe-core's label-content-name-mismatch rule is a substring-containment check, and the accessible name below contains the visible text "Cancel" as a substring even though it instructs the opposite action. A screen reader announces "Do not cancel"; a sighted user reads "Cancel". This is the WCAG 2.5.3 Label in Name failure mode axe-core cannot see, which is why 2.5.3 stays method: human_review in profiles/wcag22-aa.json even though the rule is wired into axe_tag_map for the (deterministic) fail direction.
