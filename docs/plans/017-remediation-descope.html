Backlog 017

Make Allie audit-only by deleting remediation as product surface.

Allie should audit, map, report, review, and project release decisions. It should not expose a remediation CLI, workbench step, queue schema, patch plan, or docs that imply Allie fixes findings.

Chosen Design

Delete the remediation command and generated queue path. Keep finding-level diagnostic prose and replay commands as audit evidence, but relax the schema so downstream agents can read context without Allie owning a remediation contract.

Quality System

Standards: VISION and SPEC agree; public CLI usage is coherent; workbench artifacts stop at report/review/release; Rust-first gate stays green.

Proof: focused command absence checks, workbench artifact inspection, `npm run verify`, and artifact-only critic.

Acceptance

  • No `allie remediate` command or usage line.
  • Workbench no longer creates remediation pointers or steps.
  • Smoke scripts and docs stop expecting remediation outputs.
  • Evidence schema no longer requires `suggested_remediation`.

Verification

  • cargo run --locked -- remediate returns unknown command.
  • npm run autonomous:smoke produces map/report/review/release/job receipts without remediation.
  • npm run verify passes.

Stop Rules

  • Stop if removing the field breaks existing evidence packet validation beyond the intended optionality.
  • Stop if release projection depended on remediation state for blocking decisions.
  • Do not add a replacement queue or new abstraction.

Execution

Docs first: SPEC, README, AGENTS, verification docs, and roadmap language align with audit-only. Tests next: delete/remap remediation tests and update workbench/autonomous assertions. Code last: remove parser, dispatcher, receipt types, renderers, workbench step, pointers, and queue structs.

Critic Focus

Hunt for leftover public remediation promises, broken operator workflow, schema incompatibility that is larger than ticket 017 intends, and false green verification that still checks stale remediation receipts.
