PLAN 024

Adopt clean-atomic comic-ops for Allie evidence reports.

The durable design baseline must make proof easier to inspect: hard panels, blue section tabs, dense ledgers, status slabs, and proof strips. Decoration loses to WCAG legibility and evidence clarity.

Chosen Surface

The WCAG compliance report is the first representative adoption surface because it carries the criterion ledger, proof counts, screenshots, and residual review evidence.

Design Contract

Create DESIGN.md and docs/design-contract.md with the clean-atomic token set, source provenance, and no-copy boundary for the generated concept board.

Implementation

Embed a local Misty Step-inspired token subset in src/report.rs. Generated reports stay offline and self-contained.

Stop Rule

Review required if the report loses evidence clarity, horizontal mobile fit, WCAG status legibility, or the full gate.

Verification System

Claim: representative report output visibly uses the baseline. Falsifier: no design marker, no proof strip, no hard panels, no ledger, mobile overflow, or failed canonical gate. Driver: npm run coverage:smoke, Playwright screenshots, design lint, slop detector, and npm run verify.

Source Decision

@misty-step/aesthetic is not available on npm here, and the exact 9bbe0f9 CSS URL returned 404. The reachable source is misty-step/aesthetic@v2.5.1; Allie records a local-subset adoption to protect offline report portability.
