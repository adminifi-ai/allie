ALLIE / PLAN 018

adoption-trivial slice - rendered executor contract

BACKLOG 018 - P1 - MAKE ADOPTION TRIVIAL

A consumer repo can run Allie without exporting a browser-worker path.

The first adoption slice removes the most brittle setup step: `ALLIE_BROWSER_WORKER`. The Rust CLI should locate the bundled worker from the installed binary, an adjacent checkout, or the existing explicit env override, then tell the operator how to fix missing Node or Playwright prerequisites with `allie doctor`.

CHOSEN DESIGN

Put worker discovery behind the browser-worker adapter, add `allie doctor` as the public preflight, and make consumer smoke fail if the env var is still needed.

QUALITY SYSTEM

Standards: no setup magic, no provider leakage, Rust-owned CLI contract, direct error messages, and no weakened gates.

Proof: focused Rust tests, `npm run consumer-cwd:smoke`, `npm run verify`, and thermo review on the public workflow.

Premise: Habitat work item 018, `VISION.md`, `src/worker.rs`, `scripts/consumer-cwd-smoke.sh`.

STANDARDS

Clone-or-install consumers run the same `allie verify` path locally and in CI without a hidden worker env var.

ACCEPTANCE

`allie run` from a temp consumer repo captures evidence with `ALLIE_BROWSER_WORKER` unset; `allie doctor` reports worker, Node, and Playwright readiness.

VERIFY

Consumer-CWD smoke, focused CLI tests, docs contract assertions, and full `npm run verify`.

COMMUNICATE

Report the resolved behavior, commands run, CI wrapper changes, and residual packaging limits.

STOP IF

The worker cannot be found from an installed binary without a new packaging convention, or the smoke still requires env wiring.

CURRENT STATE

`src/worker.rs` checks `ALLIE_BROWSER_WORKER`, then falls back to `env!("CARGO_MANIFEST_DIR")/workers/browser/run.mjs`. Consumer docs and CI examples export the env var.

CHANGE SHAPE

Add a small worker runtime module: resolve script candidates relative to env override, current executable, and checkout roots; expose diagnostics for `allie doctor`.

PROOF SURFACE

`scripts/consumer-cwd-smoke.sh` must delete the env override and still capture one state with zero infrastructure failures and correct Git provenance.

SCOPE

Goal: make the normal consumer path self-locating and self-diagnosing.

Non-goals: publishing a real release artifact, containers, package manager install, or solving agentic worker distribution.

Invariants: local-first execution, evidence packet as core interface, no legal compliance claim, no hidden cloud or model dependency.

REPO ANCHORS

src/worker.rs - canonical browser worker adapter.

src/consumer.rs and src/cli.rs - init, verify, and CLI usage.

scripts/consumer-cwd-smoke.sh - live consumer proof.

README.md, docs/verification.md, docs/ci/* - operator contract.

VERIFICATION SYSTEM

Claim: a consumer repo can run Allie without manually exporting the browser-worker path.

Falsifier: with `ALLIE_BROWSER_WORKER` unset, a foreign-CWD run records zero states or a worker infrastructure failure.

Driver: `npm run consumer-cwd:smoke` and `cargo run --locked -- doctor`.

Grader: evidence JSON has `states_captured >= 1`, `infrastructure_failures == 0`, and doctor reports actionable readiness.

Evidence packet: `.allie/run/latest/evidence.json`, consumer smoke transcript, and reporter docs checks.

Cadence: before implementation as red proof, after each fix, then full `npm run verify` before PR.

ALTERNATIVES

Env var forever: simplest implementation, but it preserves the adoption footgun the ticket exists to remove.

Container first: good eventual distribution, but over-scopes this slice and delays the local contract.

Node rewrite in Rust: violates the narrow Playwright boundary and is not needed for adoption.

RISKS

Installed binaries do not always know where bundled JS lives. The resolver should be explicit about searched paths and doctor should turn that into a fix, not a raw spawn failure.

Playwright dependencies still require Node setup. This slice improves discovery and guidance; it does not pretend packaging is finished.
