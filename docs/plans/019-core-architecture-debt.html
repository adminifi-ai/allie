Backlog 019

Split Allie's god-module without changing product behavior.

Move worker IPC, WCAG profile logic, CLI dispatch, and release projection out of src/lib.rs; keep existing CLI output, packet JSON, worker artifacts, and release decisions stable.

Chosen Design

Extract canonical modules that own existing concepts: worker, standards, release, and cli. No new runtime behavior, dependency, or public command.

Proof Surface

npm run verify is the repo-wide live oracle. Focused checks: worker smoke, release smoke, autonomous smoke, and size gate after each major extraction.

Stop Rules

Stop if a split needs behavior changes, if JSON output drifts unexpectedly, if lib.rs remains above the ratcheted cap, or if a module becomes a shallow pass-through.

Acceptance

  • No Command::new("node"), playwright-axe-worker, or worker response DTOs in src/lib.rs.
  • WCAG profile, criterion title/source/level/principle, and axe tag mapping live in standards.
  • CLI dispatch is separated from command handlers; command handlers remain tested through the public CLI and focused handler tests where useful.
  • Release decision projection uses typed DTOs instead of ad hoc serde_json::Value as the internal working model.
  • Size gate ratchets downward toward roughly 5200 lines and passes.

Verification System

Claim: Allie behavior is unchanged while architecture boundaries are cleaner.

Falsifier: worker/run/release/autonomous outputs break, CLI usage changes unexpectedly, or lib.rs still owns the extracted boundaries.

Driver: npm run verify plus targeted structural greps.

Grader: green gate, line-count cap, and absence checks for forbidden ownership strings in lib.rs.

Sequence

  • Extract worker DTOs, failure type, request builder, process invocation, and response reader into src/worker.rs.
  • Extract WCAG profile helpers into src/standards.rs and update compliance/report callers.
  • Move release projection and report rendering into src/release.rs, using typed release DTOs in model.rs.
  • Move CLI dispatch and per-command output handlers into src/cli.rs; keep existing option parsers and command functions in their canonical owning modules.
  • Ratcheting: update size gate only after the split is real, then close 019 with receipts.

Review Focus

Fresh reviewers should hunt for behavior drift hidden by the refactor, new thin wrappers that do not reduce cognitive load, module boundaries that merely move spaghetti, and gates that no longer exercise the moved runtime path.
