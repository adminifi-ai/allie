autonomy-depth slice - generated multi-step replay

BACKLOG 016 - P1 - DEEPEN AUDIT AUTONOMY DEPTH

Generated flows should exercise simple interactions, not just echo routes.

The first 016 milestone makes discovery produce replayable multi-step candidates through Allie's existing state-step contract. A generated flow for the workbench fixture should click the actions menu and type into settings before evidence capture, proving the audit reaches post-interaction states.

CHOSEN DESIGN

Extend `FlowCandidate` with existing `StateStep`s, infer a small deterministic interaction plan from local fixture HTML, and preserve those steps during `promote-flow`.

QUALITY SYSTEM

Standards: Rust owns schemas and promotion; the browser worker remains the only Playwright boundary; model-only findings stay advisory; no remediation scope.

Proof: focused failing-then-passing Rust test, `npm run autonomous:smoke`, full `npm run verify`, and thermo review.

Premise: `backlog.d/016-deepen-audit-autonomy-depth.md`, `src/discovery.rs`, `examples/autonomous-workbench.yml`, `fixtures/workbench/*`.

STANDARDS

Generated replay gets deeper without a second interaction DSL or hidden model dependency.

ACCEPTANCE

Flow-plan candidates include steps; promoted manifests serialize steps; run evidence proves post-action DOM states.

VERIFY

Focused Rust tests, autonomous smoke evidence, full gate, and diff review.

COMMUNICATE

Report this as child 1 of 016, with other 016 children still open unless completed.

STOP IF

Heuristics require secrets, Playwright leaks into Rust, or generated steps cannot be falsified by evidence.

CURRENT STATE

`run_discovery` emits one candidate per surface. `run_promote_flow` currently sets `steps: Vec::new()`, so generated replay is route coverage only.

CHANGE SHAPE

Carry candidate state steps through the flow-plan packet and generate conservative steps from static fixture signals like `aria-controls` buttons and email fields.

PROOF SURFACE

`npm run autonomous:smoke` checks generated flow YAML and evidence DOM artifacts for the open menu and typed email state.

SCOPE

Goal: satisfy the first 016 oracle: generated flows include multi-step interactions/assertions.

Non-goals: video-to-model ingestion, observe-act-rejudge loops, all-surface model fan-out, and precision benchmarking.

Invariant: Allie audits, maps, reports, and produces evidence; it does not remediate.

REPO ANCHORS

src/discovery.rs - discovery packet, flow plan, promotion.

src/lib.rs - manifest state-step model and tests.

workers/browser/run.mjs - existing state-step execution boundary.

scripts/autonomous-smoke.sh - live autonomous replay proof.

VERIFICATION SYSTEM

Claim: generated flows can replay deterministic interactions before evidence capture.

Falsifier: generated YAML has no `steps`, run artifacts show the menu still hidden, or typed email state is missing.

Driver: `cargo run --locked -- discover`, `promote-flow`, `run`, and `npm run autonomous:smoke`.

Grader: flow-plan JSON and generated YAML include steps; evidence DOM contains open menu and typed email values with no state-step errors.

Evidence packet: `.allie/discovery/autonomous-smoke/flow-plan.json`, `generated-flow.yml`, `.allie/runs/autonomous-smoke/evidence.json`, and DOM artifacts.
