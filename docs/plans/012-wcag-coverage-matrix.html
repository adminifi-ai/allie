P0 / WCAG 2.2 AA VISIBILITY

Make the compliance report a 55-success-criterion matrix over each required surface and state.

Chosen design: keep the evidence packet compatible, then make the report the honest visibility layer. WCAG success criteria stay the denominator; Allie deterministic, scripted, agentic, and human aggregate checks move into support sections and criterion cell provenance.

QUALITY SYSTEM

Outcome: accessibility engineers can drill from a WCAG criterion to surface/state evidence without inflated counts.

Standards: Rust-first, packet-first, no legal compliance claim, no provider leakage, no weakened deterministic gates.

Proof: matrix unit tests, schema/provenance rejection tests, CLI smoke artifacts, fresh diff critic.

Stop if report cannot distinguish WCAG criteria from support checks without breaking existing packet validation.

STANDARDS

Exactly 55 `wcag22-aa` success criteria, no duplicate criterion rows, support checks outside denominator.

ACCEPTANCE

Each required surface/state has a cell with status, applicability, method, confidence, evidence, agentic, waiver, and residual-review fields.

VERIFY

`cargo test --locked`, `npm run coverage:smoke`, then full repo gate.

COMMUNICATE

Report changed contract, exact artifacts, critic result, PR URL, final clean tree.

STOP IF

A criterion is missing, support checks re-enter denominator, model-only findings imply pass, or provenance can be empty for terminal statuses.
