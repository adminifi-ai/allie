PLAN / 015 Allie live base_url discovery

Backlog 015, child 3

Teach discovery to crawl a live base_url target, without changing Allie's audit-only product boundary.

Chosen Design

Implement a small deterministic HTTP crawler behind `src/discovery/live.rs`: start from `target.base_url`, fetch same-origin HTML, read same-origin `/sitemap.xml`, bound route count, and emit the existing discovery, flow-plan, and map artifacts.

State

`src/discovery.rs` owns fixture discovery and product-map generation. Manifests already require `target.base_url` for non-fixture targets, but discovery ignores it.

Acceptance

A local live HTTP test site with `/`, `/settings`, `/help`, and sitemap-only `/account` appears in `discovery.json`, `flow-plan.json`, and `product-map.json` when the manifest uses `kind: web` and `base_url`.

Non-Goals

No credentialed crawl, no JavaScript browser crawl, no HTTPS/TLS crawling, no agentic model change, no remediation, and no release blocking on model-only findings.

Standards

  • Rust-first; no Node or Playwright details in discovery.
  • Bounded crawl to avoid surprise network exploration.
  • Same-origin only; fragments, mailto, external hosts, non-HTTP schemes, and assets ignored.
  • Manifest-specified states remain authoritative and merge with crawl output.

Verification System

Claim: live `base_url` discovery produces routes from HTTP HTML, not only fixture files.

Falsifier: the generated artifacts miss `/settings` or sitemap-only `/account`, or include an external route.

Driver: Rust unit/CLI test with a local `TcpListener` site plus repo smokes.

Evidence Packet

Commit the regression test; leave runtime proof in `.allie/discovery/015-live-base-url-smoke/`, `.allie/maps/015-live-base-url-smoke/`, and PR body after running local commands.

Execution

1 docs

Update README/evidence docs to state that `base_url` discovery performs bounded same-origin HTTP link and sitemap discovery.

2 tests

Add a failing live HTTP discovery test using a local server and CLI artifact assertions.

3 code

Implement the non-fixture `base_url` branch through a dedicated live discovery module; keep manifest states authoritative and merge live candidates.

4 QA

Run targeted live discovery/map commands, then `cargo fmt --check`, `cargo test --locked`, `cargo clippy --locked -- -D warnings`, and `npm run verify`.

Stop Rules

  • Stop if the manifest needs new public schema fields to make the slice honest.
  • Stop if network behavior requires auth, JavaScript execution, or browser crawling for acceptance.
  • Stop if reachable live candidates fail to appear in committed tests or the persistent smoke packet.

Review Focus

Critics should hunt URL normalization bugs, unbounded crawl behavior, external-host leakage, misleading confidence labels, and whether `verify` can still imply complete coverage when the live crawl only sees static links.
