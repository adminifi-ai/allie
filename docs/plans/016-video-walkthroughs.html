ALLIE / PLAN 016

autonomy-depth slice - model video input

BACKLOG 016 - CHILD 2

The vision model should receive the recorded walkthrough clips, not only sampled stills.

Allie already records focus and motion clips for the human report. This slice moves those same WebM artifacts across the provider boundary as base64 `video_url` content when a compatible OpenRouter model call runs.

CHOSEN DESIGN

Keep Rust as the policy and packet owner. In the agentic worker, encode captured WebM clips into OpenRouter video content parts beside the existing screenshots.

QUALITY SYSTEM

Standards: no remediation scope; model-only findings remain advisory; browser capture stays in the worker; provider detail stays outside Rust.

Proof: offline fake OpenRouter smoke verifies `video_url` in the outgoing payload, plus full `npm run verify`.

CLAIM

Agentic review model requests include clips for focus or motion batches that captured clips.

FALSIFIER

The model request body contains only text and `image_url` parts, or clips remain report-only.

DRIVER

`scripts/agentic-smoke.sh` with a local HTTP server standing in for OpenRouter.

STOP IF

Video requires a hosted upload path, secrets in artifacts, or a provider-specific branch leaks into Rust.

SCOPE

Send captured focus and motion clips to the model as `video_url` base64 data URLs. Preserve existing screenshot behavior and report media.

Do not implement observe-act-rejudge, all-surface fan-out, precision benchmarking, or any remediation feature.

SOURCE CHECK

OpenRouter documents video-capable chat completions with `video_url` content parts and base64 data URLs for local videos, including WebM support.

Because provider support varies by model, errors remain degraded review context rather than release blockers.
