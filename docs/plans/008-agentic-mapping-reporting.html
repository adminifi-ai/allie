Allie / Agentic Mapping + Reporting Research + shape packet / 2026-06-19

Allie should run as an agentic mapper before it runs as a scanner.

The CLI should launch a bespoke accessibility agent in the consumer repository. That agent builds and maintains a product map, derives user stories and workflows, generates an evidence plan, executes deterministic and scripted tests through the Rust engine, sends visual artifacts through policy-gated vision review, and writes a drilldown compliance report. Remediation is explicitly out of scope for this slice.

Chosen Shape

Agent-first, engine-backed. Allie invokes an agent runner for discovery, mapping, test-plan authorship, and context enrichment; the Rust core remains the deterministic evidence, standards, policy, artifact, and report engine.

Runner posture: default to an adapter abstraction with OpenCode as the headless baseline and OMP as the richer interactive candidate. Do not bake either CLI into the product contract.

Goal

Generate a human-reviewable, evidence-backed accessibility map and WCAG report for an application with minimal manual route/story setup.

Non-Goal

No automated remediation or PR patching in this slice. The report may suggest next evidence to collect, not change application code.

Output

product-map.json, surface-map.html, generated flow manifests, evidence packets, vision review receipts, and a WCAG drilldown report.

Proof

Run against Vanity and Linejam. The map must identify real routes, visible surfaces, stories, workflows, evidence gaps, and standards status.

Stop If

The agent cannot cite repo files, route evidence, screenshots, or test artifacts for a mapped claim. Uncited map assertions are drafts, not evidence.

System Flow

1. Agent Launch

allie map starts a bounded agent in the target repo with a project-specific system prompt, budgets, allowed tools, and output schema.

2. Product Map

The agent reads code, routes, tests, docs, config, and running app evidence to produce surfaces, services, stories, workflows, and confidence.

3. Evidence Plan

Allie converts the map into flow manifests and test obligations: deterministic, scripted, agentic, human-needed, not applicable, and unknown.

4. Engine Run

Rust and the browser worker execute axe, Playwright, screenshots, DOM, accessibility tree, keyboard, video, trace, and report capture.

5. Vision Review

Policy-gated vision agents review screenshots/video for criteria that require human judgment, producing hypotheses and context receipts.

6. Visibility

Allie writes a WCAG drilldown report that starts from the standards graph and attaches evidence, gaps, and agent/human review context.

The map is the central artifact for agentic discovery. It must be reviewable by an accessibility compliance engineer and machine-usable by Allie.

Report Contract

The report should start from every WCAG obligation, then drill into applicability, product surfaces, evidence, findings, screenshots, videos, tests, agent context, and human review status.

Initial Run

allie map --agent opencode --out .allie/map

Launches the mapper agent, creates the first product map, generates candidate flows, and writes a human-readable surface map.

Report Run

allie report --map .allie/map/product-map.json --out .allie/report

Runs evidence collection and produces the standards drilldown report. This is the main slice for compliance visibility.

Diff Run

allie report --since origin/main --map .allie/map/product-map.json

Limits required evidence to changed surfaces while preserving the full standards ledger and marking stale or missing coverage honestly.

Accuracy Model

Status Meaning Allowed evidence
Pass The obligation was applicable and sufficient evidence supports compliance for mapped surfaces. Deterministic test, scripted test, human attestation, or high-confidence agentic context only when policy permits that status.
Fail Evidence shows non-compliance or required behavior broke. axe/static failure, scripted failure, or reviewed human/agent finding promoted by policy.
Needs Review Allie has enough context to guide a human but not enough to claim pass/fail. Vision review, screenshot/video, DOM, a11y tree, content context, or manual checklist prompt.
Not Tested No credible evidence exists yet. Report must show missing driver, missing artifact, or unmapped surface. No hidden gray areas.
Not Applicable The obligation does not apply to the mapped surface. Applicability rule, agent/human rationale, and surface scope evidence.

Implementation Slices

1. Agent runner adapter: define a small interface: prompt, cwd, budgets, allowed tools, session id, transcript export, JSON artifact path.

2. Product-map schema: surfaces, routes, components, services, user stories, workflows, evidence refs, confidence, human review flags.

3. Mapper prompt: a bespoke accessibility-compliance engineer prompt that requires citations to files, routes, screenshots, and commands.

4. Standards graph: ingest W3C/WAI references into local profiles with source URLs and update metadata.

5. Report generator: standards-first HTML/JSON drilldown over product map plus evidence packets.

6. Vision review lane: policy-gated artifact batches, prompts, redaction receipts, and non-blocking context enrichment.

Verification System

Claim: Allie can autonomously map a consumer app and generate a standards-first report a compliance engineer can review.

Falsifier: the map misses visible surfaces, uncited assertions appear in the report, or WCAG statuses cannot be traced to evidence.

Driver: allie map and allie report against Vanity and Linejam.

Grader: schema validation, golden report snapshots, seeded-missing-route fixture, target-identity guard, and human review of the first generated surface map.

Evidence packet: .allie/dogfood/<repo>/agentic-map-report/ with transcript, product map, screenshots, generated flows, evidence packet, vision receipts, and report HTML.

Cadence: every runner adapter change and every standards/report schema change.
