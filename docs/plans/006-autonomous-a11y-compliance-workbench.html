Backlog 006-010

Autonomous Accessibility Compliance Workbench

Allie should let a compliance engineer point at an application and receive autonomous discovery, generated browser evidence, full WCAG drilldown, agentic vision context, remediation guidance, and release enforcement without pretending model opinion is legal compliance.

Premise source: current operator direction in this thread, codified into docs/vision.md and backlog items 006 through 010.

Repo baseline: branch deliver/full-backlog has a clean V0 loop: handwritten manifest, Playwright/axe worker, screenshot, packet, local report, and release projection.

Key correction: the LLM, vision AI, comprehensive WCAG matrix, autonomous discovery, and remediation workbench are not shipped. They are now explicit epics.

Decision

Build a packeted autonomy pipeline, not a semantic agent platform. Rust owns schemas, policy, budgets, hashing, promotion state, and enforcement. Browser and model systems are narrow adapters that return typed evidence. Every generated or agentic claim must replay, be attested, or stay neutral.

Target Operator Experience

Assess

Discover sitemap, product surfaces, user stories, route states, hidden UI, form errors, media, and auth boundaries under explicit budgets.

Prove

Generate Playwright and axe coverage, replay it, capture DOM, accessibility tree, screenshots, video/GIF, traces, console, network, and raw axe JSON.

Explain

Render every WCAG criterion with status, method, rationale, artifacts, model context, waiver state, remediation, and residual review.

Pipeline

Discovery packet
Sitemap, surfaces, states, provenance, budgets.
Story packet
User stories, critical flows, confidence, operator review.
Flow plan
Generated Playwright candidates, WCAG methods, replay intent.
Evidence packet
Axe, scripted checks, screenshots, DOM, a11y tree, media, traces.
Review packet
Vision AI attempts, prompts, responses, redaction receipts, confidence.
Release packet
Blocking proof, neutral review gaps, waivers, remediation, check payload.

External Anchors

Standards

  • W3C WCAG overview: WCAG success criteria determine conformance, so Allie's denominator must be criteria, not tool rules.
  • W3C WCAG-EM: evaluation has scope, exploration, representative samples, evaluation, and reporting.
  • W3C ACT: test methods should be transparent across automated, semi-automated, and manual testing.

Automation

  • Deque axe API docs: axe tags map rules to WCAG versions and success criteria.
  • Deque also notes hidden regions need to be activated or rendered before analysis, which points directly to generated Playwright interaction coverage.

Stop Conditions

  • Stop if generated flow candidates can block release before replay.
  • Stop if model-only findings can block release without scripted reproduction or human attestation.
  • Stop if reports imply legal compliance instead of evidence status and residual review.
  • Stop if provider calls can run without typed policy, redaction receipts, budgets, and no-fallback enforcement.

Verification Contract

Autonomy is accepted only when it leaves evidence. Every new capability must produce a reviewable artifact set: schema validation, packet, report, replay command, raw worker/model transcript, artifact hashes, release projection, and final gate output. A green unit test without those receipts is not product acceptance.
