ALLIE / PLAN 016

autonomy-depth slice - vision fail precision

BACKLOG 016 - CHILD 5

Vision FAIL verdicts should promote only after a zero-false-positive labeled gate.

Agentic review can be useful evidence, but a model-only false FAIL is expensive. This slice adds a labeled precision gate to the worker response and makes Rust refuse agentic FAIL promotion unless the gate reports zero false-positive failures. PASS promotion and inconclusive honesty stay unchanged.

CHOSEN DESIGN

The worker evaluates optional labeled cases against per-surface model results and emits `precision_gate`. Rust treats `precision_gate.status=pass` with zero false positives as the only condition that allows FAIL promotion.

QUALITY SYSTEM

Substantive agent-policy change: fake-provider precision smoke, Rust promotion unit tests, full `agentic:smoke`, autonomous smoke, full verify, and thermo review.

CLAIM

Agentic FAIL cannot promote unless a labeled precision gate proves zero false-positive FAILs for the model/run.

FALSIFIER

A fake model fails a labeled known-pass surface and Rust still promotes the FAIL, or the benchmark passes with a false positive.

DRIVER

`npm run agentic:precision`, focused Rust tests, `npm run agentic:smoke`, `npm run autonomous:smoke`, then `npm run verify`.

STOP IF

The gate requires live external labels, blocks releases on model-only findings, or weakens the existing pass/inconclusive behavior.
