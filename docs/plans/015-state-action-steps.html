Backlog 015 / child 1

Let manifest states perform deterministic setup actions before evidence capture.

Allie already logs in through auth.steps. This slice adds non-secret flow.states[].steps so a state can click, fill, type, and wait before axe, screenshot, DOM, accessibility-tree, keyboard, video, and trace evidence are captured.

Chosen Design

Add a small state-step DTO beside ManifestState and execute it inside the browser worker after goto and auth-marker verification, before evidence capture. Login secrets stay in auth.steps.

Proof Surface

npm run action:smoke drives the checked-in fixture, opens hidden UI, and proves the DOM artifact contains the revealed panel. npm run verify keeps the full regression floor.

Stop Rules

Stop if action values become secret-bearing, if failures disappear into a clean packet, if auth login and state actions share an unclear schema, or if the worker request drifts from the evidence contract.

Acceptance

  • Manifests accept flow.states[].steps with fill, type, click, and wait_for.
  • The worker executes steps before capture, so DOM/screenshot/axe evidence reflects the post-action state.
  • Failed state steps record a state error and block instead of crashing or passing silently.
  • Docs distinguish non-secret state setup from secret-bearing auth.steps.
  • The full repo gate remains green.

Verification System

Claim: a manifest can open hidden UI before Allie records evidence.

Falsifier: the DOM artifact lacks the revealed panel, or a missing selector still produces a pass packet.

Driver: npm run action:smoke plus focused Rust validation tests and npm run verify.

Evidence packet: .allie/runs/action-steps-smoke/ with worker request, evidence packet, report, and DOM artifact.

Review Focus

Critics should hunt schema confusion between auth and state steps, secret leakage through literal action values, state-step failure modes that look like clean evidence, and any Playwright detail leaking beyond the worker boundary.
